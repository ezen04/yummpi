# Github code convention

## 커밋 메시지 형식

`<태그>(<scope>): <제목>`

- 태그는 소문자
- 콜론(`:`) 뒤 띄어쓰기 한 칸
- 제목은 한 줄, 50자 이내 권장
- 마침표 없음
- "수정함", "작업" 같은 모호한 표현 지양 — 무엇을 했는지 명확하게

---

## 태그 목록

| 태그 | 설명 | 예시 |
| --- | --- | --- |
| **feat** | 새로운 기능 추가 | `feat: 로그인 기능 구현` |
| **fix** | 버그 및 기존 기능 수정 | `fix: 회원가입 시 비밀번호 검증 오류 수정` |
| **refactor** | 코드 리팩터링 (기능 변화 없음) | `refactor: UserService 코드 구조 개선` |
| **style** | 코드 스타일·포맷팅 변경 | `style: Home 페이지 세미콜론·들여쓰기 통일` |
| **docs** | 문서, 주석 수정 | `docs: README 설치 방법 업데이트` |
| **chore** | 환경 설정, 의존성 설치·삭제 등 | `chore: eslint 설정 추가` |
| **test** | 테스트 코드 추가·수정 | `test: 로그인 E2E 테스트 추가` |

---

## Scope (범위 명시)

태그 뒤 괄호에 작업 도메인을 명시한다. 선택 사항이지만 5인 팀에서 히스토리를 볼 때 어느 영역의 변경인지 바로 파악할 수 있어 권장한다.

```jsx
feat(auth): 카카오 로그인 구현
feat(voting): 실시간 투표 동기화 구현
fix(settlement): 정산 금액 소수점 오류 수정
refactor(map): Haversine 계산 함수 분리
chore(ocr): CLOVA OCR API 키 env 등록
```

| Scope | 해당 도메인 |
| --- | --- |
| `auth` | 인증·로그인 |
| `gathering` | 모임 생성·관리 |
| `map` | 장소 추천·지도 |
| `voting` | 실시간 투표 |
| `settlement` | 정산 |
| `payment` | 송금·알림 |
| `dashboard` | 진입 대시보드·마이페이지 |
| `infra` | 환경 설정·배포·CI |
| `ocr` | 영수증 업로드·OCR·검수 |
| `shared` | 공용 타입(Zod 스키마 등 공용 유틸) |
| `design` | 클라이언트 화면 디자인 생성 및 수정 |

## 커밋 단위 원칙

**하나의 커밋에 하나의 변경사항만** 담는다.

```jsx
# ❌ 한 커밋에 여러 작업
git commit -m "feat: 투표 기능 구현 + 버그 수정 + 스타일 변경"

# ✅ 작업 단위로 분리
git commit -m "feat(voting): 투표 집계 로직 구현"
git commit -m "fix(voting): 동률 처리 시 주최자 선택 미동작 오류 수정"
git commit -m "style(voting): 투표 결과 화면 간격 조정"
```

---

## 좋은 예시 vs 나쁜 예시

```jsx
# ❌
git commit -m "수정"
git commit -m "작업중"
git commit -m "fix"
git commit -m "aaa"
git commit -m "일단 커밋"

# ✅
git commit -m "feat(auth): 카카오 소셜 로그인 구현"
git commit -m "fix(settlement): OCR 실패 시 fallback 미동작 오류 수정"
git commit -m "refactor(map): 중간 지점 계산 함수 Haversine으로 교체"
git commit -m "chore: pnpm workspace 모노레포 초기 설정"
```

## 작성 예시

```jsx
git commit -m "feat(gathering): 모임 생성 API 연동"
git commit -m "fix(voting): 실시간 투표 동기화 누락 오류 수정"
git commit -m "refactor(settlement): 정산 계산 로직 함수 분리"
git commit -m "chore: tailwind v4 의존성 추가"
git commit -m "docs: git-flow 브랜치 전략 문서 추가"
git commit -m "test(ocr): 영수증 업로드 E2E 테스트 추가"
```
